# 分支系统与互动引擎架构设计

## 一、核心公式

```
角色状态向量 × 剧情节点 × AI视频生成 = 千人千面的互动体验
```

同一个节点，不同角色状态，看到不同的视频，拥有不同的选项。

## 二、角色状态向量

```yaml
character_state_vector:
  visual:        # 直接影响视频画面
    level: int
    equipped_title: str
    outfit: str
    weapon_skin: str
    aura: str
    mount: str
  capability:    # 决定可用选项
    attributes: {str, agi, int, vit, cha, luk}
    skills: []
    passive_skills: []
    combat_power: int
  relational:    # 决定NPC反应和隐藏选项
    npc_relationships: {}
    faction_standing: {}
  world_state:   # 改变世界的状态
    flags: {}
    economy: {}
    weather: str
```

## 三、视频生成管线

```
节点模板 + 角色状态向量
    ↓
状态感知的Prompt Engine
    ├→ 视觉层注入：角色外观、装备、光环
    ├→ 场景层注入：flags改变的环境状态
    ├→ 叙事层注入：关系值影响的NPC行为
    ↓
融合Prompt → 关键帧 → 视频 → 选项注入 → 播放
```

## 四、Git式内容分叉模型

        [玩家A分支]  [玩家B分支]  [玩家C分支]

## 五、分支创建流程——"在游玩中创作"

```
游玩中到达某个节点
    ↓
点击"创造新的可能性"
    ↓
输入想法（一句话）
    ↓
AI生成分支预览（视频+剧情+选项）
    ↓
确认 → 发布分支
```

## 六、五个核心体验循环

1. **探索循环**（分钟级）：观看视频 → 选择 → 结果 → 反馈 → 想看更多
2. **成长循环**（小时级）：探索 → 经验/物品 → 升级 → 解锁 → 更强
3. **创作循环**（天级）：游玩 → 灵感 → 分支 → 发布 → 反馈/收益
4. **社交循环**（周级）：发现角色卡 → 用TA视角游玩 → 合作/竞争 → 社群
5. **经济循环**（月级）：创作 → 上架 → 收益 → 更多创作

## 七、技术架构

### 新增服务
- **分支管理服务**：分叉/合并/可视化/差异存储/版本控制
- **AI导演服务**：状态感知提示词引擎/叙事一致性/分支推荐

### 数据库扩展
- `world_branches` — 世界分支表
- `world_state_snapshots` — 世界状态快照
- `world_dynamic_variables` — 动态世界变量

## 八、用户体验黄金路径

```
第1分钟：游客身份进入热门世界 → 看AI视频
第3分钟：第1个选项 → 选择 → 看到不同结果
第5分钟：自动创建临时角色 → 获得第1件装备
第7分钟：引导创建正式角色卡 → 自定义外观/名字
第10分钟：提示"可以创造分支" → AI一键生成分支 → 用户看到自己创造的内容
注册转化完成 → 用户建立情感绑定
```

## 九、竞争力壁垒

| 维度 | 传统互动视频 | 传统游戏 | 本平台 |
|------|-----------|--------|------|
| 内容生产 | 专业团队 | 专业开发 | AI+UGC+分支网络 |
| 内容丰富度 | 有限 | 有限 | 无限（AI+分支） |
| 角色相关性 | 固定主角 | 固定场景 | 角色状态影响一切 |
| 创作门槛 | 高 | 极高 | 游玩即创作 |
| 网络效应 | 低 | 低 | 极高 |
